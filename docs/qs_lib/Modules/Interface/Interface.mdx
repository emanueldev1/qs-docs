# Interface

## Changing the Primary Color

You can customize the primary color of the UI to align with your server's theme. This change affects elements such as the progress bar/circle, skill check, radial menu center button and hover, dialog confirm buttons, input field focus, and more. Adjusting the primary color requires only setting the appropriate convars and restarting the resourceâ€”no UI rebuild is necessary.

Available preset colors include options like `blue`, `red`, `green`, `yellow`, `cyan`, `pink`, `indigo`, `purple`, `teal`, `orange`, and `gray`. Each color has shades ranging from 0 (lightest) to 9 (darkest), allowing fine-tuned customization.

## Convars

Set the primary color and shade using the following convars in your `server.cfg`:

```yaml
setr qs:primaryColor blue
setr qs:primaryShade 8
```

### Examples

#### Example 1: Setting a Vibrant Theme

Configures the UI with a vibrant teal color and a medium-dark shade.

```yaml
setr qs:primaryColor teal
setr qs:primaryShade 6
```

#### Example 2: Using a Dark Theme

Sets the UI to a dark purple theme with the darkest shade.

```yaml
setr qs:primaryColor purple
setr qs:primaryShade 9
```

## Custom Palette Guide

  <AlertTitle>Advanced Customization</AlertTitle>

If you want to create a custom color palette, you can define your own colors and shades. This involves modifying the UI's source code to include your palette, which can then be used in place of the preset colors. After defining your palette, rebuild the UI to apply the changes.

### Examples

#### Example 1: Defining a Custom Palette

Sets a custom palette with a unique color name and shade range.

```yaml
setr qs:primaryColor customEmerald
setr qs:primaryShade 5
```

#### Example 2: Using a Custom Neon Theme

Applies a custom neon color palette with a bright shade.

```yaml
setr qs:primaryColor neonPink
setr qs:primaryShade 3
```
