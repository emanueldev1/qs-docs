# Player Bridge (Client)

<Alert variant="info" className="mb-4">
  <AlertTitle>Default Compatibility</AlertTitle>
  <AlertDescription>
    The functions listed below are compatible by default with the following frameworks: QBX-Core, QB-Core, and ESX. These functions represent the complete set of client-side player operations provided by this bridge. Note that some functions, such as <code>lib.player.getGang</code>, may not be fully supported in all frameworks. In such cases, the function may return default values if gang systems are not implemented.
  </AlertDescription>
</Alert>

The Player Bridge provides functions to manage player-related data on the client side, such as retrieving identity, inventory, job, and status information.

## lib.player.identifier

Retrieves the player's unique identifier.

```lua
lib.player.identifier()
```

**Returns:**

- identifier: `string` | `nil`
  - The player's identifier (e.g., "license:abc123" or "citizenid123"), or `nil` if not available.

### Example: Getting Player Identifier

Fetches and logs the player's unique identifier.

```lua title="Client: Getting Player Identifier"
RegisterCommand('getid', function(source, args, rawCommand)
    local identifier = lib.player.identifier()
    if identifier then
        print("Player identifier:", identifier)
    else
        print("Player identifier not available.")
    end
end, false)
```

## lib.player.getPlayerData

Retrieves the player's data or a specific key from it.

```lua
lib.player.getPlayerData(_key)
```

- _key: `string` | `nil`
  - Optional key to retrieve specific data (e.g., "job").

**Returns:**

- data: `table` | `any`
  - The player's data or the value of the specified key (e.g., `{ job = { name = "police" }, money = { cash = 100 } }` or `{ name = "police" }` for `_key = "job"`), or `nil` if not available.

### Example: Getting Player Data

Fetches and logs the player's job data.

```lua title="Client: Getting Player Data"
RegisterCommand('getjobdata', function(source, args, rawCommand)
    local jobData = lib.player.getPlayerData("job")
    if jobData then
        print("Player job:", jobData.name, "Grade:", jobData.grade)
    else
        print("Player job data not available.")
    end
end, false)
```

## lib.player.hasItem

Checks if the player has a specific item in their inventory.

```lua
lib.player.hasItem(item, amount)
```

- item: `string`
  - The item name (e.g., "water").
- amount: `number` | `nil`
  - The minimum amount required (default: 1).

**Returns:**

- hasItem: `boolean`
  - `true` if the player has the item and amount, `false` otherwise.

### Example: Checking for an Item

Checks if the player has at least 2 units of "water".

```lua title="Client: Checking for an Item"
RegisterCommand('hasitem', function(source, args, rawCommand)
    local hasItem = lib.player.hasItem("water", 2)
    print("Player", hasItem and "has" or "does not have", "at least 2 water items.")
end, false)
```
